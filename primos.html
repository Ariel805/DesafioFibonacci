Contexto del Problema

Los pagos digitales, las tarjetas y muchas comunicaciones seguras usan métodos de cifrado. Uno de los más conocidos es RSA, que se apoya en números primos muy grandes porque multiplicarlos es fácil, pero descubrir sus factores puede ser extremadamente difícil.

En la vida diaria también usamos códigos numéricos como un PIN. Este verificador es educativo: no convierte un PIN en una contraseña perfecta, pero muestra cómo una idea matemática como la primalidad puede combinarse con otras reglas para detectar códigos demasiado obvios.

Explicación del Algoritmo

Un número primo es aquel que solo tiene dos divisores positivos: 1 y él mismo. Para verificar si un número es primo, no hace falta probar todos los divisores; basta con revisar hasta su raíz cuadrada.

2 3 5 7 11 13 17 ... 
function esPrimo(numero) {
  if (numero < 2) return false;

  for (let divisor = 2; divisor * divisor <= numero; divisor++) {
    if (numero % divisor === 0) {
      return false;
    }
  }

  return true;
}

El simulador evalúa si el PIN es primo y además revisa patrones inseguros: secuencias, números repetidos, códigos comunes y variedad de dígitos.

Conclusión

Los números primos son importantes porque permiten construir sistemas de seguridad como RSA. En este ejemplo, la idea se simplifica para analizar un PIN: un número primo es mejor que uno con patrones evidentes, pero la seguridad real depende de que el PIN sea secreto, difícil de adivinar y no esté relacionado con fechas personales.
